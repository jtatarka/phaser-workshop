Put Your Game Face On

Introduction

  1. Phaser

    2. From its Web site, "Phaser is a fun, free and fast 2D game framework for making HTML5 games for desktop and mobile web browsers, supporting Canvas and WebGL rendering."

    In other words, Phaser is a toolbox that developers can use to create all kinds of online games. It provides a starting point so that we can begin designing and building without having to write a lot of foundational code.

  2. HTML5

    3. HTML (HyperText Markup Language) is the most basic building block of the Web. Using tags, HTML specifies the content and structure of a webpage which web browsers use to render its substance.

    HTML5 is the latest standard that defines HTML which aims to allow developers to create more advanced Web sites. Phaser exploits the HTML5 <canvas> tag to create images to be used for online games.

  3. CSS

    4. CSS (Cascading Style Sheets) is used to style Web pages. Rather than focusing on content like HTML, CSS specifies how a Web page should look using various definitions which are applied using selectors.

    We won't be using CSS heavily in this workshop, however its good to know for context.

  4. JavaScript

    5. JavaScript is the de facto scripting language of Web pages. Though lightweight, it is extremely powerful and capable of controlling the behavior of Web pages or running many non-browser environments.

    We'll be programming our games in this workshop using JavaScript and the Phaser framework.

Working with Thimble

Thimble is an online editor created by Mozilla which can be used to learn HTML, CSS, and JS. We'll use Thimble to create and host our Phaser games within the Cloud (meaning it's available from any computer with no need to install any software) for free.

Creating an Account

Follow these directions to get setup with a Thimble account of your own:

  1. First, go to thimble.mozilla.org and click on "Create an account".
  2. Enter your desired username, a valid email address, and a password.
  3. Create a new project and get started with help from the default template, or remix this project which includes the boilerplate code needed to start a new Phaser 2 game.

Remixing

Thimble allows you to remix any project that is published on thimbleprojects.org. This means you can take a copy of the code, edit it within your own account, and republish a new version of the same application. To remix a project, just click on the "Remix" button from any Thimble site.

Publishing

When you're ready to share your work, from any Thimble project's editor, click on "Publish" in the upper right-hand corner. Enter a brief description of your project, then click on "Publish" again and your application will be created and hosted. Once completed, you'll be presented with a link that you can share with family and friends.

Previewing

When working on a site using Thimble it may or may not be useful to have the preview auto refresh. When working directly with HTML and CSS this works really well, however with some JS applications, such as ones made with Phaser, it might be better to manually refresh after you've completed a method or other update. You can toggle between these two modes by checking or unchecking "AUTO" at the top of the editor.

Debugging

As with any coding project, you're bound to run into some issues and bugs while you're developing your Phaser game. Luckily, Phaser has console logging which helps us identify issues in code, and Thimble has a built in console to display these messages when using the editor. You can reveal it at anytime by clicking on the console icon at the bottom right of the preview panel.

For more advanced debugging, you can use Chrome's Developer Tools (press F12 to open, then click on the Sources tab). Dev Tools lets you set breakpoints in your code and watch variables at different times during code execution to help identify, troubleshoot, and fix bugs.

A Bit More Detail

Although Phaser makes it easy for us to jump right in and get started with building online games, there is a little bit of setup required. First, we need to create a basic HTML page for our game to live in, maybe set a few basic styles, and definitely import the Phaser library, which is packaged up nicely in a .min.js file.

After remixing the boilerplate project, you'll see the basic HTML structure for our application in index.html. HTML is uses tags to organize the content of a Web page, and a barebones page will be formatted like this:

<html>

  <head>
    <!-- Header content goes here including metadata, CSS and JS imports -->
    <link rel="stylesheet" href="style.css">
    <script src="https://cdn.jsdelivr.net/phaser/2.2.2/phaser.min.js"></script>
    <!-- Everything in here is not shown in the browser's view -->
  </head>

  <body>
    <!-- Content to be displayed goes here, structured using various HTML tags -->
    <!-- Our body is empty other than a script we'll write to create our game -->
  </body>

</html>

The <head> above includes a script tag to import phaser.min.js and a link to connect our CSS file. You might notice that all of the tags above have an associated tag of the same type with a / before it to close out that block of code, except for link. Most tags in HTML require both an opening and closing tag, though not all do, and those are known as self-closing.

The linked styles.css includes all of our CSS rules which the browser will use to style our page. Take a look at this file and you'll see this:

body {
  margin: 0;
}

All CSS rules are formatted like this. First we have a selector, in this case body and then any number of styles to be applied to the elements in our HTML file which match the selector. In this case a margin of 0 will be applied to our entire page.

This selector selects on a particular element directly, but we can also match ids and classes. Any one HTML page should only have one of any given unique id, but may have any number of elements with a given class applied. For example, look at this HTML snippit:

<span id="welcome">Welcome to My Web Page!</span>

<p class="orange">This is a paragraph with some information.</p>

To select the id we need to use a # before the name, and for classes we use a . like this:

#welcome {
  font-weight: bold;
}

.orange {
  color: orange;
}

But the boilerplate project sets all of this up for us and we won't be focused on either HTML or CSS for this workshop. The important part is the Phaser library, which is written in JS. We'll be writing our own JS script using Phaser's code to help us out.

JavaScript is technically a multi-paradigm language, meaning it can support different styles of coding. Often JS libraries take an object oriented (OO) approach, and this includes Phaser. The Phaser library specifies many different objects, or classes, each with their own variables and methods. Variables can also be described as properties, and methods as functions. In the games we create we'll see lines of code like this:

player.animations.add('left', [0, 1, 2, 3], 10, true);

In this code, player is an object of type Phaser.Sprite, which has a property called animations, which happens to be another object (nesting these two is called composition) of type Phaser.AnimationManager. Then Phaser.AnimationManager has a method called .add() which contains the logic to add an animation to the manager of the Sprite. You can diferentiate properties from methods by looking for parentheses.

Any things within the parenteses of a method are called parameters, but note that some methods do no require parameters. In the example above, .add() takes four parameters of various types, which are used within the logic of the method. The first is a string (a collection of characters strung together, the next an array (a data structure), then an integer, and finally a boolean (true or false).

On a very high level, to make a Phaser game we'll first create a new Phaser.Game object, and then fill in three of Phaser's essential methods: .preload(), .create(), and .update().

The .preload() function is used to load any assets the game needs, like images, before building the entire game. Next, .create() creates the actual pieces of the game which will be displayed in the viewport. Then finally, .update() is a loop which is called on repeat to rerender the screen. We'll add interactions to this method and tell Phaser what we want the game to do when different things happen.

You'll see each of these topics in more detail throughout the workshop. But one additional thing to note about Phaser: it's on a coordinate plane that might be orientied differently than expected. The x-axis is positive in the right (east) direction, and the y-axis is positive in the downward (south) direction.

Getting Started with Phaser

To get started, we'll be follwing an adaptation of the official Phaser tutorial "Making your first Phaser game", which is available directly from phaser.io. Note that we'll be using Phaser CE (Community Edition; also known as Phaser 2) for this workshop, but Phaser 3 has recently been released and has the same tutorial for the new, completely rewritten framework available here. It's still a work in progress, however.

Step 0 - Boilerplate

All Phaser games start by creating an instance of the Phaser.Game object, typically called game by convention. You can call this varible anything you'd like, but can you really think of a better name for it?

Remix this project, or copy the code below into an existing Web page to get started with any new Phaser game.

var game = new Phaser.Game(800, 600, Phaser.AUTO, '', { preload: preload, create: create, update: update });

function preload() {

}

function create() {

}

function update() {

}

Line 1 creates this local variable called game and assigns the object to it for later use. The new keyword creates a new object with the given parameters, which are the five variables inside the parentheses.

The first two parameters are the width and height of your game. We're using 800 and 600 by default, but you can create a game of any size you like.

The third parameter is the rendering context. It's a good idea to always use Phaser.AUTO as it will determine which option is best for the user's system: either Phaser.WEBGL or Phaser.CANVAS, which are the only other two valid choices for this parameter.

Next up is the element of the Web page in which your game will live. By specifying an empty string ('') we'll append to the page's <body>. Finally, the fifth parameter is an object which maps our references to Phaser's essential functions.

Step 1 - Loading Assets

The preload() function is automatically called by Phaser when the game starts. The application will look to this function and load any assets, such as sprites and sprite sheets, which are defined within it.

We will load our assets to our game using game.load and the .image() and .spritesheet() methods, as appropriate for each type of asset:

function preload() {

    game.load.image('sky', 'assets/sky.png');
    game.load.image('ground', 'assets/platform.png');
    // ...
    game.load.spritesheet('dude', 'assets/dude.png', 32, 48);

}

The code above adds three images to our game and one spritesheet. The first two parameters specify the asset key, used to identify the asset in subsequent lines of code, and the resource location.

After loading the 'ground', add one more image named 'star' from the resource 'assets/star.png'.

Note that .spritesheet() has two additional parameters: the width and height of each frame in the sheet, plus an optional fifth parameter which will tell Phaser how many frame to use from the sheet if there are any empty frames at the end.

Step 2 - Create a Sprite

Now that we've loaded some assets, let's turn them into a sprite so we can use them in our game. Sprites should be added to the create() function of our application, and will look like this:

function create() {
        
  game.add.sprite(0, 0, 'sky');
  
}

Now you should have a nice gradient that will serve as the background for our game! Behind the scenes, Phaser created a new Phaser.Sprite object and added it to the game world, where all the objects will live.

Step 3 - World Building

The game world in Phaser is infinite and expands in all directions on a coordinate plane. The origin, (0, 0), is automatically placed at the upper left corner of a new game for reference. We'll add some platforms to our newly created world that are character can run and jump on.

First, create a new variable called platforms. Then, within the create() functions add in the Physics system and initialize a group which will contain the ground and two ledges:

var platforms;
        
function create() {

  // Add the Arcade Physics system to the game world
  game.physics.startSystem(Phaser.Physics.ARCADE);

  game.add.sprite(0, 0, 'sky');

  // Initialize a group called platforms
  platforms = game.add.group();

  // Enable physics for all objects within the platforms group
  platforms.enableBody = true;

}

To create the ground and ledges, create new variables and use platforms.create() to add them to the group in the correct position. Note that we can use game.world.height to get the height of the game world, like this:

var ground = platforms.create(0, game.world.height - 64, 'ground');

ground.scale.setTo(2, 2);

ground.body.immovable = true;

Following the same format, add two ledges to the platforms group. One should be on the right of the viewport, and the other on the left. When finished you'll have a world in which your character can move around.

Step 4 - Sprite Sheets

Our sprite sheet from Step 2 was loaded with a different method because sprite sheets are not images. They differ because a sprite sheet is made up of animation frames, not a single picture. If you didn't take a look at the 'dude' asset before, here's what he looks like:

You'll notice that the sprite sheet has nine frames: one in the middle for when he's not moving and facing the camera, and two sets of four frames to either side for when he's running to the left and right.

To add him to the game world, create a new variable called player, then reference this code:

// Add the player and its settings
player = game.add.sprite(32, game.world.height - 150, 'dude');

// Enable physics on the player
game.physics.arcade.enable(player);

// Set the player's physics properties
player.body.bounce.y = 0.2;
player.body.gravity.y = 300;
player.body.collideWorldBounds = true;

// Add two animations to get him walking to the left and right
player.animations.add('left', [0, 1, 2, 3], 10, true);
player.animations.add('right', [ ... ], 10, true);

First, we'll add a sprite with three parameters. To see what these parameters are, take a look at the official Phaser documentation. Next, enable Arcade physics on the player and set the related properties: bounce, gravity, and collideWorldBounds.

Finally, we'll add two animations which will make him move. The first one is completed for you, and the parameters, in order, are: a name for the animation, the frames from the sprite sheet to use in the animation, the desired frameRate, and a boolean which signifies that the animation should be repeated on a loop. This method is also fully documented here.

Can you complete the 'right' animation with the correct frames?

Step 5 - More on Physics

We've already specified that our game will use the Physics.ARCADE system and enabled it on all of our sprites and groups. For the playforms we used enableBody, and for our player we enabled arcade, and doing so gave our objects an instance of ArcadePhysics.Body.

For example, when creating our player, we added the line player.body.gravity.y = 300;, which sets the local gravity, or how fast he'll fall. 300 is an arbitrary number, but the higher this value, the faster he'll come down while jumping or falling from platforms.

But you might have noticed that when he falls he's not landing on the ground, but rather on the edge of the world. Try commenting out the line with collideWorldBounds and see what happens to our dude.

Physics.ARCADE also allows us to check collisions of any body in the game world. Add back the line we just commented out so he can't leave the frame, and then insert the following line into the update() function:

function update() {

  var onPlatform = game.physics.arcade.collide(player, platforms);
  
}

This function is called by the game engine for every frame and the .collide() function will check to see if the player and platforms have collided, or touched. If they have, it will stop them from overlapping, and this is how we can get the dude to land on the ground and ledges without falling to the bottom of the game world.

Step 6 - Keyboard Controls

Great, now our dude is able to land correctly, but he also can't yet move, so let's fix that! One way we can control characters using Phaser is with arrow keys. To do this, all we have to do is add a new variable and create a cursors object.

var cursors;

function create() {

  // ...

  cursors = game.input.keyboard.createCursorKeys();

}

Calling .createCursorKeys() returns an object with four properties for each arrow key: up, down, left, and right. With just one line of code we can easily update() the location of our player.

Now that we have access to keyboard controls, we need to add in some logic to move the 'dude' around:

function update() {

  // ...

  // Reset his velocity in the x-axis
  player.body.velocity.x = 0;

  // Conditional statements to move him side to side
  if (cursors.left.isDown) {

    // Move him to the left
    player.body.velocity.x = -150;
    player.animations.play('left');

  } else if (cursors.right.isDown) {

    // Move him to the right
    player.body.velocity.x = 150;
    player.animations.play('right');

  } else {

    // Make him stand still
    player.animations.stop();
    player.frame = 4;

  }

Awesome, now our little 'dude' can move around to the left and to the right. But can he hop onto one of the ledges? Nope, not yet. If you're thinking we can use cursors.up to make him jump, you're absolutely right!

Let's add a new conditional check to make him jump, as follows:

if (cursors.up.isDown && onPlatform && player.body.touching.down) {

  // Make him jump up
  player.body.velocity.y = -350;

}

The second and third checks here make sure that he is touching one of the platforms, as we (probably) don't want him to get an extra boost when he's already jumping or on top of some other object.

Step 7 - Collecting Stars

Mr. 'dude' seems a little lonely, doesn't he? Let's give him a purpose! We'll drop some stars into the game world and let him collect them. Just as with the platforms, we can add a group() and then create() several objects in it.

var stars;

function create() {

  // ...

  stars = game.add.group();

  stars.enableBody = true;

  // Create 12 stars as part of the stars group
  for (var i = 0; i < 12; i++) {

    // Space them evenly (70px) apart
    var star = stars.create(i * 70, 0, 'star');
  
    // Apply gravity and a slightly random bounce
    star.body.gravity.y = 300;
    star.body.bounce.y = 0.7 + Math.random() * 0.2;
  
  }

If you run the code, you might notice a problem. Just as we did with the player and platforms, we need to .collide() the stars and platforms so the stars don't fall through.

Next we can have our 'dude' collect the stars using the .overlap() method:

game.physics.arcade.overlap(player, stars, collectStar, null, this);

You can probably tell what player and stars mean here, but what about collectStar? This third parameter is a callback function which gets invoked when the two previous parameter overlap. The fouth parameter, null is another callback function that lets you make additional checks before calling the first callback, and finally this is a reference to the context. Don't worry so much about the latter parameters; the first three are most important for us, but if you want to learn more, check out the documentation.

But more on that callback function; we need to create it! Outside of the existing three functions, add:

function collectStar(player, star) {

  // Remove the star from the game world
  star.kill();

}

A note about the parameters here: Phaser is smart enough to automatically send them to our callback function since it knows we're checking if the player and star have overlapped.

Step 8 - Keeping Score

So now we have a fully functional game, but let's take it one set further and keep score based on how many stars the player has collected. We can do this two two extra variables: one to keep track of the score, and one to display it:

// Initalize score to zero
var score = 0;

// Initalize Phaser.Text object to display the score
var scoreText;

Then let's setup the scoreText (think about where this should go):

scoreText = game.add.text(16, 16, 'Score: 0', { fontSize: '32px', fill: '#000000' });

If you're wondering what the first two parameters here mean, change them up and see what happens. The third parameter is the string of text we'll be adding, and the last one inside { } is an object used to style the text.

After fully initializing scoreText we need to update it when a star is collected. We can piggy back off of our callback function to do this since both actions will happen at the same time:

function collectStar(player, star) {

  // ...

  score += 10;
  scoreText = 'Score: ' + score;

}

And there you have it, a fully functional Phaser game! Test it out and make sure your score is correctly incremented with each star that you collect.

Making It Your Own

Add Your Personal Style

Put the finishing touches on your game by updating its look!

Do a Google image search for a new background or texture to use for the platforms, or search through some of the sites in the References below. Upload your new images to Thimble and update your JS code to use them.

You can also style the rest of your Web page, since a plain white background is boring. First, add a new HTML element to your page with an id. We'll use this element to append the game to, rather than <body>:

<div id="game"></div>

Then update your game variable with the same id as the constructor's fourth parameter. From here, you're ready to apply CSS styling to the game's viewport. Just setup some rules in styles.css using the id as your selector:

#game {
  border: 10px solid white;
  margin: 50px auto;
  width: 800px;
}

Change the Collected Item

The assets folder contains lots of additional sprites you can add to your game. Try to change the collected item from a star to a diamond or one of the colored balls. Note that these assets have different widths and to make them evently spaced out you might need to adjust the location of where they get created. Also, you can add more than 12 items - try doubling that number or making a hundred!

Bonus: use all of the ball assets in your for loop to create items of different colors. Can you come up with a good way to iterate though each color to minimize the amount of duplicated code?

Go Crazy with Platforms

Make your game more challenging by adding additional ledges to the platforms group. Put them at different locations around the game world and use scale.setTo() to adjust the height and width of each one. With the proper parameters, can you turn a ledge into a pillar that the player will need to jump over?

Also try setting body.immovable = false; and see what happens to the ground or platforms. If you remove some stars you may be able to allow the player to only jump on a ledge once.

Take It to the Next Level with Sound

Images and sprite sheets aren't the only kind of assets that Phaser will let you load; you can also add sound to your games as background music or effects when some action occurs. The boilerplate project contains a few clips to get you started, including one called key.wav, which can be added to your game like this:

var sound;

function preload() {

  // ...

  game.load.audio('sound', 'assets/key.wav');

}

function create() {

  // ...

  sound = game.add.audio('sound');

}

Then your sound is ready to use whenever you need it with just one additional line:

sound.play();
If you wanted to play this sound when an item was collected, where could this line be added to the game?

Pick a New Sprite Sheet

From a game design perspective, it's really important to carefully think about the characters you want to introduce into your game world. Picking the right ones will give a unique feel to your game and have a big impact on your users' overall experience.

Included in the Phaser Boilerplate project is another sprite sheet called puppy.png that we can use:

This sprite sheet is more complicated than dude.png for a few reasons: there are more frames to choose from, including a blank one at the end, and they're all facing to the right. But we can make it work!

First, let's change our code from 'dude' to 'puppy'. We'll need to change the last line of preload(), to rename the asset, its location, and the size of each frame.

function preload() {
  
  // ...

  game.load.spritesheet('puppy', 'assets/puppy.png', ..., ...);

}

Open the asset in the Thimble editor to see the size of the sprite sheet. How can you determine the height and width of each frame with this information?

Next, update the player declaration in create() so we reference the correct asset. It just so happens that using frames [5, 6, 7, 8] works well for this sprite sheet too, so let's try it out and see what happens.

With just those few changes, making the 'puppy' move to the 'right' works quite well, but things get weird when we go to the 'left'. Update the frames in the 'left' animation to match those in the 'right' and we get a pretty close. But he's still not facing the correct direction.

How can me make the 'puppy' not moonwalk? Luckily Phaser can help us flip the sprite sheet. To do this, first add an anchor, and then adjust the scale:

function update() {

  // ...

  player.anchor.setTo(0.5, 0.5);

  if (cursors.left.isDown) {
    player.scale.x = -1;
    // ...
  } else if (cursors.right.isDown) {
    player.scale.x = 1;
    // ...
  }

  // ...

}

Setting the anchor places an imaginary point in the middle of the sprite sheet that Phaser will use to flip the image over, creating an axis in both dimentions. Then, when the user presses the left arrow key we'll flip the image, and likewise flip it back to the original orientation when the right arrow key is pressed down.

One final improvement could be to update the default frame to number 5, but can you think of ways to use some of the other frames that are in the sprite sheet?
